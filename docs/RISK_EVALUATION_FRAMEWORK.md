# Workspace守り番 リスク評価フレームワーク

## 1. 概要

本ドキュメントは、Google Driveファイルの共有設定に基づくリスク評価の体系を定義します。
ISMS（ISO27001）およびプライバシーマークの観点を考慮した設計となっています。

---

## 2. リスク評価の基本方針

### 2.1 評価対象

Google Drive内でユーザーがアクセス可能なすべてのファイルを評価対象とします。
ただし、**オーナーの所属（社内/社外）** と **自社の関与度** によってリスクの重み付けを変更します。

### 2.2 リスクの分類

| リスク種別 | 説明 | 対象 |
|-----------|------|------|
| **漏洩リスク** | 自社の情報が外部に漏洩するリスク | 自社所有ファイル |
| **受領リスク** | 外部から受け取った情報の管理不備リスク | 外部所有ファイル |

### 2.3 ISMSとの整合性

- ISMSでは「自社の適用範囲内で取り扱う情報」はすべて情報資産として管理対象
- 外部から共有されたファイルも、自社がアクセス権を持つ限り管理対象に含まれる
- ただし、リスクの**性質**と**重大度**は自社所有ファイルとは異なる

## 3. ファイル分類マトリクス

### 3.1 オーナー × 公開設定 × 自社関与度

```
┌─────────────────────────────────────────────────────────────────────────┐
│                         ファイル分類マトリクス                           │
├─────────────┬─────────────────┬─────────────────┬─────────────────────┤
│   オーナー   │   公開設定      │   自社の関与     │   リスク分類        │
├─────────────┼─────────────────┼─────────────────┼─────────────────────┤
│   自社      │ 公開リンク      │ -               │ 漏洩リスク（最高）   │
│   自社      │ 外部ユーザー共有 │ -               │ 漏洩リスク（高）     │
│   自社      │ 社内のみ        │ -               │ 漏洩リスク（低）     │
├─────────────┼─────────────────┼─────────────────┼─────────────────────┤
│   外部      │ 公開リンク      │ 編集者          │ 受領リスク（中）     │
│   外部      │ 公開リンク      │ 閲覧者のみ      │ 受領リスク（低）     │
│   外部      │ 公開リンク      │ 関与なし        │ 対象外（0%）        │
│   外部      │ 限定共有        │ 編集者          │ 受領リスク（中）     │
│   外部      │ 限定共有        │ 閲覧者のみ      │ 受領リスク（低）     │
└─────────────┴─────────────────┴─────────────────┴─────────────────────┘
```

### 3.2 リスク係数

| 分類 | 係数 | 理由 |
|------|------|------|
| 自社オーナー | **100%** | 自社の情報漏洩に直結 |
| 外部オーナー + 自社が編集者 | **30%** | 社内情報を書き込む可能性あり |
| 外部オーナー + 自社が閲覧者のみ | **10%** | 管理台帳には載せるが実質低リスク |
| 外部オーナー + 自社関与なし | **0%** | 評価対象外 |

## 4. リスク要因と配点

### 4.1 基本リスク要因

| # | リスク要因 | 基本点数 | 深刻度 | 説明 |
|---|-----------|---------|--------|------|
| 1 | 公開リンク共有 | **50点** | Critical | 「リンクを知っている全員」がアクセス可能 |
| 2 | 外部ユーザー共有 | 20点 | High | 組織外のユーザーとファイルを共有 |
| 3 | 外部編集権限 | **25点** | High | 組織外のユーザーが編集可能 |
| 4 | 機密ファイル形式 | 15点 | Medium | スプレッドシート、PDF等の機密性が高い形式 |
| 5 | 機密ファイル名 | 10-25点 | Low-Critical | ファイル名に機密情報を示唆するキーワード |
| 6 | 長期未更新の共有 | 10点 | Low | 1年以上更新のない共有ファイル |
| 7 | 多数ユーザー共有 | 5点 | Low | 10人以上と共有 |

> **変更履歴 (v1.1)**:
> - 公開リンク共有: 40点 → 50点（単独でHigh、機密情報と組み合わせでCriticalに到達させるため）
> - 外部編集権限: 15点 → 25点（編集権限付与は閲覧より重大なリスク）

### 4.2 追加リスク要因（外部ファイル向け）

| # | リスク要因 | 点数 | 深刻度 | 説明 |
|---|-----------|------|--------|------|
| 8 | 外部ファイルへの社内編集参加 | 10点 | Medium | 外部所有ファイルに自社メンバーが編集権限を持つ |

### 4.3 スコア計算式

```
最終スコア = Σ(基本点数 × オーナー係数) + 追加リスク点数

※ 上限100点
```

## 5. リスクレベル判定

| スコア範囲 | レベル | ラベル | 推奨アクション |
|-----------|--------|--------|---------------|
| 80-100点 | Critical | 緊急 | 即座に対応が必要 |
| 60-79点 | High | 高 | 早急に確認・対応 |
| 40-59点 | Medium | 中 | 計画的に対応 |
| 0-39点 | Low | 低 | 定期レビュー時に確認 |

## 6. 具体例

### 6.1 自社所有ファイルの例（Critical）

**ケース**: 自社メンバーがオーナーの「顧客リスト.xlsx」を「リンクを知っている全員」に公開

| 要因 | 基本点 | 係数 | 最終点 |
|------|--------|------|--------|
| 公開リンク共有 | 50 | 100% | 50 |
| 機密ファイル形式 | 15 | 100% | 15 |
| 機密ファイル名 | 25 | 100% | 25 |
| **合計** | | | **90点（Critical）** |

### 6.2 自社所有ファイルの例（High）

**ケース**: 自社メンバーがオーナーのスプレッドシートを「リンクを知っている全員」に公開

| 要因 | 基本点 | 係数 | 最終点 |
|------|--------|------|--------|
| 公開リンク共有 | 50 | 100% | 50 |
| 機密ファイル形式 | 15 | 100% | 15 |
| **合計** | | | **65点（High）** |

### 6.3 自社所有ファイルの例（Medium）

**ケース**: 自社メンバーがオーナーのドキュメントを外部ユーザーに編集権限付きで共有

| 要因 | 基本点 | 係数 | 最終点 |
|------|--------|------|--------|
| 外部ユーザー共有 | 20 | 100% | 20 |
| 外部編集権限 | 25 | 100% | 25 |
| **合計** | | | **45点（Medium）** |

### 6.4 外部所有ファイルの例（自社が編集者）

**ケース**: 外部ベンダーが所有するドキュメントに自社メンバーが編集者として参加、公開リンク設定

| 要因 | 基本点 | 係数 | 最終点 |
|------|--------|------|--------|
| 公開リンク共有 | 50 | 30% | 15 |
| 外部ファイルへの社内編集参加 | 10 | - | 10 |
| **合計** | | | **25点（Low）** |

### 6.5 外部所有ファイルの例（閲覧のみ）

**ケース**: 取引先が共有してきた資料（閲覧のみ）、公開リンク設定

| 要因 | 基本点 | 係数 | 最終点 |
|------|--------|------|--------|
| 公開リンク共有 | 50 | 10% | 5 |
| **合計** | | | **5点（Low）** |

## 7. フィルタリング機能

ユーザーが効率的にリスクを確認できるよう、以下のフィルターを提供します。

### 7.1 所有者フィルター

| フィルター | 説明 |
|-----------|------|
| すべて | 全ファイルを表示 |
| 自社所有のみ | 自社メンバーがオーナーのファイルのみ |
| 外部所有のみ | 外部ユーザーがオーナーのファイルのみ |

### 7.2 リスクレベルフィルター

| フィルター | 説明 |
|-----------|------|
| Critical | 緊急対応が必要なファイル |
| High | 高リスクファイル |
| Medium | 中リスクファイル |
| Low | 低リスクファイル |

### 7.3 推奨ビュー

**デフォルト表示**: 「自社所有のみ」+ 「Medium以上」
→ 実際に対処すべきリスクに集中できる

## 8. 将来の拡張案

### 8.1 検討中の追加リスク要因

| 要因 | 説明 | 検討状況 |
|------|------|---------|
| ファイル内容のキーワード検出 | 本文内の機密情報検出 | API制限により困難 |
| アクセスログ分析 | 異常なアクセスパターン | Audit Log API連携で実装可能 |
| 共有履歴の変更検知 | 権限の急な変更 | 実装予定 |

### 8.2 ISMS/Pマーク対応レポート

- 情報資産台帳の自動生成
- リスクアセスメント結果のエクスポート
- 是正措置の追跡機能
